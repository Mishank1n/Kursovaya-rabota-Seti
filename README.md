## 📋 О проекте

Веб-приложение для мониторинга состояния устройств в локальной сети. Система автоматически сканирует диапазон IP-адресов, определяет доступные узлы через ICMP (ping), собирает данные о задержке и потерях пакетов, а также отображает всё это в удобном веб-интерфейсе.

---

## ✨ Функциональность

- 🔍 **Автоматическое сканирование** — определяет текущий сегмент сети и параллельно опрашивает все адреса диапазона `/24`
- 📡 **Ping-диагностика** — измеряет RTT и процент потерь пакетов для каждого узла
- 🔄 **Периодический опрос** — фоновое обновление статусов устройств по расписанию без участия пользователя
- 🗄️ **История сканирований** — каждый запуск фиксируется в базе данных с временными метками и статистикой
- 🖥️ **Веб-интерфейс** — таблица устройств с live-графиком ping, статусами ONLINE/OFFLINE, MAC-адресами и типами устройств
- 🌐 **ARP-интеграция** — сопоставление IP-адресов с MAC-адресами через системную ARP-таблицу

## 📊 Как работает сканирование

```
1. ArpService определяет локальный IP и читает ARP-таблицу
2. NetworkScanService вычисляет базовый адрес и формирует диапазон .1–.254
3. PingService параллельно (CompletableFuture) отправляет ICMP-пакеты
4. Доступные узлы сохраняются/обновляются в H2 со статусом ONLINE
5. Недоступные узлы помечаются OFFLINE (но не удаляются)
6. Создаётся запись ScanRun с итоговой статистикой
7. @Scheduled-методы повторяют опрос автоматически
```

## 📸 Интерфейс

Веб-интерфейс отображает:

- Параметры сети (Gateway, Current IP, Subnet, количество устройств)
- Таблицу найденных устройств с колонками: IP · MAC · Тип · Статус · Средний Ping · Packet Loss
- Live-график RTT для каждого устройства (по клику)
- Время последнего обновления

## 📚 Использованные протоколы

**ICMP** — проверка доступности узлов и измерение RTT  
**ARP** — сопоставление IP и MAC-адресов  
**REST / HTTP** — взаимодействие между серверной частью и веб-интерфейсом
